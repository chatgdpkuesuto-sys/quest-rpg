# 開発フロー（鉄則）

## 原則
- **一度に一つのイベント（フェーズ）しか作らない**
- ボタンを押して「進む」のではなく、ボタンを押した「結果を見て」進める
- TRPGのように：プレイヤーが意志を示す → GM（AI）が解釈 → 画面に反映
- セリフは毎回VOICEVOXで読ませる
- 更新前に必ずセーブする。リロードでリセットは絶対NG

## 各フェーズの作り方

### 1. コードを書く（4ファイルのみ）
- `bridge.py` — そのフェーズの通信＋処理ロジックだけ
- `00_Dashboard/index.html` — そのフェーズの画面だけ
- `00_Dashboard/js/engine.js` — そのフェーズのJS処理だけ
- `00_Dashboard/style.css` — そのフェーズのスタイルだけ

### 2. テストする
- ブラウザで動作確認
- ボタン → bridge受信 → 結果表示 → ボイス再生

### 3. テンプレートを保管する
- 動いたら `00_Core_Engine/templates/` にコピー保存
- セーブデータは `99_Save_Data/` に保存

### 4. 次のフェーズへ
- 4ファイルを全部消して、ゼロから次のフェーズだけを書く
- 前のフェーズの結果（セーブデータ）は読み込める
