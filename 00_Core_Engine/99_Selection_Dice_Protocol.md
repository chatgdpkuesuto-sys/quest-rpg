# 🎲 Heroine Selection Dice Protocol (v1.0)

このテンプレートは、物語の開始時や新章突入時に「3人のヒロイン」を提示し、ダイス目によって運命の相手を決定する際に使用せよ。

## 1. 候補の選定 (Selection of Candidates)
- 現在の拠点または状況に応じたヒロインを **3名** 選出する。
- 例: [Aria, Zena, Elara]

## 2. ダイス・マッピング (Dice Mapping)
ダイス（2d6 の結果を1d3に変換、または 1d6 相当で計算）に基づき、以下の比率で配分せよ。

| ダイス目 (1d6) | 選択されるヒロイン | 運命の呼び声 |
|:---:|:---|:---|
| **1 - 2** | **Candidate 1** | 「導かれし光」 |
| **3 - 4** | **Candidate 2** | 「狂おしき本能」 |
| **5 - 6** | **Candidate 3** | 「甘美なる深淵」 |

## 3. 出力形式 (Output Protocol)
AIは `situation_text` において、以下のステップで描写を行うこと。

1. **候補の紹介**: 3人のヒロインが目の前に（あるいは運命の交差点に）現れる様子を簡潔に描写。
2. **ダイスロールの宣言**: 「運命のダイスが振られたわ…」等のメタ・ナレーション。
3. **判定結果**: ダイス目（`ai_hub.py` から提供される `dice_res`）に基づき、どのヒロインとのイベントが発生するかを決定。
4. **個別エンカウント開始**: 決定したヒロインとの情熱的な出会いや、捕食/誘惑シーンへと移行。

## 4. UI生成ルール (UI HTML)
`ui_html` には、選ばれたヒロインとの「最初の対峙」にふさわしい選択肢を生成せよ。
- 例: 「受け入れる」「抵抗する」「服従を誓う」等。
