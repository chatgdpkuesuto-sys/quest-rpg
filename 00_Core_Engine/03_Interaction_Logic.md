# インタラクション・ロジック (判定処理フロー)

プレイヤーの行動をどのようにルールに落とし込み、結果を出力するかのガイドライン。

## 1. アクションの解釈 (Interpretation)
プレイヤーの入力（自然言語）をシステムコマンドに変換する。

- **「攻撃する！」** → `/check Attack` (Str/Dex vs AC)
- **「説得してみる」** → `/check Persuasion` (Cha vs Wis)
- **「様子を見る」** → `/check Perception` (Wis vs DC)
- **「逃げる」** → `/check Athletics` (Str vs Opposed) or Command `/flee`

## 2. 難易度(DC)の設定 (Setting DC)
GMは状況に応じてDCを決定する。`04_Rule_System.md` 参照。

| 状況 | DC | 例 |
|:---|:---|:---|
| **確実** | なし | 歩く、話す、食事する。 |
| **簡単** | 10 | 鍵のかかっていないドアを開ける、友好的な相手との会話。 |
| **普通** | 15 | 鍵開け、警戒している相手の説得。 |
| **困難** | 20 | 複雑な罠の解除、敵対的な相手の説得。 |
| **超難関**| 25+ | 神業、洗脳された相手の説得。 |

## 3. 判定実行 (Roll)
ダイスロールを行い、結果を判定する。

- **クリティカル(20)**: 「劇的成功」。追加情報やボーナスを与える。
- **ファンブル(1)**: 「劇的失敗」。反撃を受ける、アイテムが壊れるなど。

## 4. 結果の描写 (Narration)
数字の結果を物語に変換する。**「Yes, and...」 (成功) / 「No, but...」 (失敗)** の精神で。

- **成功時**:
    - プレイヤーの意図通りに事が進む。
    - *「あなたは剣を振るい、ゴブリンの盾ごと腕を切り飛ばした！」*
- **失敗時**:
    - 意図は達せられなかったが、状況は変化する。
    - *「説得しようとしたが、彼は聞く耳を持たない。逆に怒り出し、剣に手をかけた！」*

## 5. 状態変化の適用 (Update Status)
描写に合わせて内部データを更新する。
- HPの減少。
- Sanity / Loyalty の変動。
- アイテムの消費・入手。
